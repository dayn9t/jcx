<task type="auto">
  <name>Task 2: Create .env.example template</name>
  <files>.env.example</files>
  <action>
Create `.env.example` with documented environment variables:

```bash
# Environment Configuration Template
# Copy this file to .env and fill in your values
# NEVER commit .env to version control

# =============================================================================
# Logging Configuration
# =============================================================================

# Log level: DEBUG, INFO, WARNING, ERROR, CRITICAL
# Default: INFO
LOG_LEVEL=INFO

# Log format: json (production) or text (development)
# Default: text
LOG_FORMAT=text

# =============================================================================
# Database Configuration (Optional)
# =============================================================================

# PostgreSQL connection URL
# Required if using database features
# DATABASE_URL=postgresql://user:password@localhost:5432/jcx

# =============================================================================
# Redis Configuration (Optional)
# =============================================================================

# Redis connection URL
# Required if using Redis caching or pub/sub
# REDIS_URL=redis://localhost:6379/0

# =============================================================================
# MQTT Configuration (Optional)
# =============================================================================

# MQTT broker URL
# Required if using MQTT messaging
# MQTT_URL=tcp://localhost:1883

# MQTT client ID (optional, defaults to random)
# MQTT_CLIENT_ID=jcx-client

# =============================================================================
# HTTP Configuration (Optional)
# =============================================================================

# HTTP proxy for outbound requests
# HTTP_PROXY=http://proxy.example.com:8080

# HTTPS proxy for outbound requests
# HTTPS_PROXY=http://proxy.example.com:8080

# =============================================================================
# Development Settings (Optional)
# =============================================================================

# Enable debug mode
# DEBUG=false

# Python path (usually set automatically)
# PYTHONPATH=/path/to/project
```

Notes:
- All variables are commented out by default (not required for basic usage)
- Each section is clearly labeled
- Variables include descriptions of purpose and defaults
- Database, Redis, MQTT, and HTTP sections are optional
- Logging is the only "always available" configuration
</action>

  </verify>
